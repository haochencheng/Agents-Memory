## 错误上下文

**任务目标：**
Add a doctor command to verify whether a project is fully integrated with Agents-Memory

**出错文件 / 位置：**
- `scripts/memory.py`

## 错误描述

The initial project resolution logic treated `spec2flow` as a relative filesystem path before checking registered project IDs, so running `amem doctor spec2flow` inside the Spec2Flow repo resolved to `/Users/cliff/workspace/Spec2Flow/spec2flow` and produced false FAIL results.

## 根因分析

The resolver prioritized `Path(project_id_or_path).resolve()` over the registry lookup, which breaks when a project ID is also a valid relative directory name in the current working directory.

## 修复方案

Changed `_resolve_project_target()` to prefer exact registered project IDs first, then fall back to path resolution only when no registered project matches.

## 提炼规则

When a CLI accepts both logical IDs and filesystem paths, resolve explicit registered IDs before probing relative paths to avoid cwd-dependent behavior.
