## 错误上下文

**任务目标：**
Protocol Fundamental Brief 从 Postgres 加载历史快照时抛出 Pydantic ValidationError。

**出错文件 / 位置：**
`provider_service/app/services/protocol_fundamental_brief/repository.py`

## 错误描述

```
pydantic.ValidationError: ... field 'serviceId' missing / extra fields forbidden
```

旧快照以 snake_case 字段名（`service_id`, `freshness_sec`, `generated_at`）持久化，
新版 Pydantic schema 使用 camelCase alias，加载旧记录时直接报字段不存在。

## 根因分析

Pydantic v2 默认只接受 alias 字段名（camelCase），
旧数据以原始 Python 字段名（snake_case）存储，形成数据格式与模式版本不兼容。
加载历史快照时没有做兼容性适配层。

## 修复方案

在 repository 层加入兼容性验证：尝试标准解析，失败时回退到 `by_alias=True, by_name=True`
联合容错解析，能读旧格式又不破坏新格式。

## 提炼规则

持久化 Pydantic 模型时，若将来可能改 alias 或字段名，
要么在序列化时固定字段名（禁用 alias），要么在反序列化层加兼容性适配。
不要假设历史数据与当前 schema 格式永远一致。
