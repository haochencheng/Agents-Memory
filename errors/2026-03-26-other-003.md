## 错误描述

A multi-hunk patch partially applied to adjacent docstring/help blocks, leaving command lines with inconsistent indentation even though the code still compiled.

## 根因分析

The patch targeted small repeated lines inside a long usage block, so context drift caused formatting-only corruption that was easy to miss in the initial diff.

## 修复方案

I replaced the entire contiguous usage block and re-read the affected sections after patching, then re-ran py_compile and a command smoke test.

## 提炼规则

When editing long CLI/docstring help blocks with apply_patch, replace the full contiguous block and re-read it once instead of stacking small line-level patches.
