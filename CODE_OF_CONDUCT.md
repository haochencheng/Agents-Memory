# Code of Conduct

## Purpose

Agents-Memory is built for serious engineering collaboration. Treat discussions as part of delivery work: stay respectful, evidence-based, and focused on improving the repository.

## Expected Behavior

1. Discuss ideas and changes in good faith.
2. Give concrete reproduction steps, validation output, or code references when raising concerns.
3. Assume collaborators are trying to improve the project, even when proposals differ.
4. Prefer precise technical disagreement over personal criticism.
5. Respect privacy and avoid posting secrets, private project data, or sensitive local paths.

## Unacceptable Behavior

1. Personal attacks, harassment, intimidation, or discriminatory remarks.
2. Repeatedly derailing technical threads with hostility or bad-faith argument.
3. Publishing private information, credentials, or non-public project context without permission.
4. Abusive language, sexualized content, or threats of violence.

## Scope

This applies to issues, pull requests, code review, discussions, and other repository-linked collaboration spaces.

## Reporting

If you encounter a conduct problem, contact the repository maintainers through a private GitHub channel when possible. Do not use a public issue for personal safety or privacy-sensitive reports.

## Maintainer Responsibilities

Maintainers may edit, hide, lock, or remove content that violates this code of conduct, and may restrict participation when needed to protect the project and its contributors.
