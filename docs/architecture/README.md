# 架构决策与分析

本目录存放架构设计文档，回答"为什么这样设计、技术取舍是什么"。遵循 ADR（Architecture Decision Record）规范。

## 文档

- [架构总览（overview.md）](overview.md)：仓库级 ADR（ADR-001 ~ ADR-006）与技术取舍，不重复产品定位。
- [模块化结构（modular.md）](modular.md)：代码目录结构、模块分层（runtime / services / commands / integrations）、插件扩展点。
- [架构分析报告（analysis.md）](analysis.md)：当前架构优化方向、CLI/MCP 集成矩阵、重构追踪。

## 读者

工程师，以及需要理解代码结构决策的贡献者。
