# BUG-002: Wiki functions passed AppContext instead of wiki_dir Path

**Date:** 2026-03-28
**Severity:** P1 (blocks all ingest operations)
**Status:** Fixed

## Description

`ingest.py` called `list_wiki_topics(ctx)`, `append_timeline_entry(ctx, ...)`, and
`update_compiled_truth(ctx, ...)` passing the `AppContext` object directly.

All three functions in `services/wiki.py` expect `wiki_dir: Path` as their first argument,
not an `AppContext`.

## Error

```
AttributeError: 'AppContext' object has no attribute 'exists'
```

Traceback pointed to `list_wiki_topics` line 321:
```python
if not wiki_dir.exists():
```

## Root Cause

The implementation of `services/ingest.py` followed `wiki_compile.py`'s pattern where
wiki functions accept `ctx`, but wiki functions in `wiki.py` were designed to
accept `wiki_dir: Path` directly for composability.

## Fix

Changed all three call sites in `services/ingest.py`:

```python
# Before (wrong)
known_topics = list_wiki_topics(ctx)
append_timeline_entry(ctx, topic, entry)
update_compiled_truth(ctx, first_topic, compiled_truth_update)

# After (correct)
known_topics = list_wiki_topics(ctx.wiki_dir)
append_timeline_entry(ctx.wiki_dir, topic, entry)
update_compiled_truth(ctx.wiki_dir, first_topic, compiled_truth_update)
```

## Test Added

`tests/test_ingest_service.py` — 33 tests cover all ingest paths including
`test_updates_wiki_timeline_on_success` which exercises the fixed call site.

## Lesson

When calling wiki service functions, always check the first argument signature:
- `wiki.py` functions take `wiki_dir: Path` (composable, path-based)
- `wiki_compile.py` functions take `ctx: AppContext` (context-based, multi-service)
