## Bug 描述

`build_compiled_page` 中当通过 `existing_frontmatter` 参数传入时，`updated_at` 字段没有被正确刷新。

## 失败信息

```
AssertionError: 'updated_at: 2026-04-07' not found in rebuilt page
```

## 根本原因

测试通过 `_SAMPLE_V2_PAGE.split("---\n\n")[0].rstrip()` 截取 frontmatter，实际得到的字符串只包含第一个 `---` 到内容前的部分（不包含闭合的 `---`），因此 `_refresh_updated_at` 找不到第二个 `---` fence，无法匹配更新。

`_frontmatter_end` 需要完整的两个 `---` fence 才能工作，而 `split("---\n\n")[0]` 仅保留开头的一个 `---` 行。

## 修复方案

修改测试，使用正确的 frontmatter 字符串（包含闭合 `---`）；  
同时在 `build_compiled_page` 中增加防御：若 `existing_frontmatter` 缺少闭合 fence 时自动补全。
