# BUG-005: plans 根目录分组创建后未补齐 planning bundle 必需文件

## 发现场景

`python3.12 scripts/memory.py plan-check .`

## 失败信息

```text
[FAIL] plan_files missing required file: docs/plans/active/README.md
[FAIL] plan_files missing required file: docs/plans/active/spec.md
[FAIL] plan_files missing required file: docs/plans/archive/README.md
[FAIL] plan_files missing required file: docs/plans/deprecated/spec.md
...
```

## 根本原因

`plan-check` 会把 `docs/plans/` 下的每个子目录都视为一个 planning bundle。
在新增 `active/`、`archive/`、`deprecated/` 分组目录后，只补了普通说明文档，
没有补齐 planning bundle 约束要求的根文件：

- `README.md`
- `spec.md`
- `plan.md`
- `task-graph.md`
- `validation.md`

因此目录结构虽然对人类可读，但对治理工具来说仍是无效 bundle。

## 修复方案

为 `active/`、`archive/`、`deprecated/` 三个根分组目录补齐最小 planning bundle 文件，
并确保每个文件都包含 `plan-check` 所要求的语义短语。

## 状态

已修复。
