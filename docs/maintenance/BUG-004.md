# BUG-004: ingest 日志仍使用 `datetime.utcnow()`

## 发现场景

`tests/test_ingest_service.py::TestBuildLogEntry::test_all_types_can_build_log_entry`

## 失败信息

```text
DeprecationWarning: datetime.datetime.utcnow() is deprecated and scheduled for removal in a future version.
Use timezone-aware objects to represent datetimes in UTC: datetime.datetime.now(datetime.UTC).
```

## 根本原因

`agents_memory/services/ingest.py::_build_log_entry` 仍使用：

```python
datetime.utcnow().isoformat() + "Z"
```

虽然当前逻辑仍能工作，但它生成的是 naive UTC 时间，已经被 Python 标记为废弃路径。
该问题会在持续测试中产生 warning，也会让日志时间语义不够明确。

## 修复方案

改为显式使用 timezone-aware UTC：

```python
datetime.now(UTC).isoformat().replace("+00:00", "Z")
```

这样仍保留现有 `Z` 结尾格式，同时消除废弃 warning。

## 状态

已修复。
