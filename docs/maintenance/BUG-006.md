# BUG-006: 项目接入后 project API 仍返回全局 wiki 统计，导致接入可见性失真

## 发现场景

在实现 Synapse-Network 自动接入时，项目页需要根据接入后的 wiki / ingest 数据判断是否成功。

## 问题表现

- `/api/projects` 中每个项目的 `wiki_count` 都返回全局 wiki 总数
- `/api/projects/{id}/stats` 未返回 `ingest_count` / `last_ingest` / `last_error`
- `/api/wiki` 未暴露 `project` 元数据，前端无法按项目筛选 wiki 页面

## 根本原因

项目 API 只聚合了全局 wiki 目录，没有基于 wiki frontmatter 与 ingest log 的 `project` 字段做 per-project 汇总。
因此实际已经接入并导入知识的项目，在前端仍然表现为“统计不准确”或“看不到关联 wiki”。

## 修复方案

- 为 wiki 页面补充 `project` / `source_path` frontmatter
- `/api/projects`、`/api/projects/{id}/stats` 改为按项目聚合 wiki / ingest / error 数据
- `/api/wiki` 与 graph 节点暴露 `project` 元数据，供前端项目详情页过滤展示

## 状态

已修复。
