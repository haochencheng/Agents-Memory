# BUG-008: 真实项目 onboarding 会把 `.pytest_cache` 文档当知识源并把本地 wiki/log 运行数据暴露到 git 工作区

## 发现场景

对 `Synapse-Network` 执行真实 `/api/onboarding/bootstrap` 后，Agents-Memory 仓库本地产生了两类噪音：

- 导入了 `services/gateway/.pytest_cache/README.md` 这类临时缓存文档
- `memory/wiki/` 与 `memory/ingest_log.jsonl` 被真实运行写脏并出现在 git 变更中

## 根本原因

1. `discover_project_wiki_sources()` 的忽略集合没有覆盖 `.pytest_cache`
2. `.gitignore` 没有忽略真实运行产生的 `memory/wiki/` 和 `memory/ingest_log.jsonl`

## 修复方案

- 将 `.pytest_cache` 纳入知识源发现忽略列表
- 将 `memory/wiki/` 与 `memory/ingest_log.jsonl` 纳入 `.gitignore`
- 补回归测试，确保缓存目录不会被纳入自动摄取

## 状态

已修复。
