# BUG-007: `amem mcp-setup` 遇到已有旧 agents-memory 条目时不会修复，导致 doctor 持续告警

## 发现场景

在对 `Synapse-Network` 进行真实自动接入验证后，`amem doctor .` 仍然报告：

- `agents-memory exists but args do not include /Users/cliff/workspace/agent/Agents-Memory/scripts/mcp_server.py`

## 问题表现

- 目标项目的 `.vscode/mcp.json` 已经包含 `agents-memory` server 条目
- 但该条目保留了历史路径或过期参数
- 执行 `amem mcp-setup .` 后，命令直接跳过，不会把旧条目修复为当前路径

## 根本原因

`write_vscode_mcp_json()` 只要检测到已有 `agents-memory` 条目就直接返回，未比较现有条目是否与期望配置一致。

## 修复方案

- 若 `agents-memory` 条目已存在且内容与期望 server entry 不一致，则覆盖修复
- 保留其他 MCP server 条目不变
- 为“已有正确条目时跳过”和“已有旧条目时修复”分别补回归测试

## 状态

已修复。
