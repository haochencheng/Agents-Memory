# BUG-003: cmd_wiki_lint 不识别 `--check=value` 格式

## 发现场景

`tests/test_mcp_phases.py::TestMCPWikiLint::test_fresh_page_no_stale_warning`

## 失败信息

```
AssertionError: 'fresh-topic' unexpectedly found in "⚠️  发现 1 个问题:\n\n  • [orphan] 'fresh-topic' 无入链（无其他页面引用它）\n"
```

## 根本原因

`mcp_app.py::memory_wiki_lint` 向 `cmd_wiki_lint` 传递参数时使用了等号格式：

```python
cmd_wiki_lint(ctx, [f"--check={check}"] if check != "all" else [])
# → ["--check=stale"]  （单个 token，含等号）
```

但 `cmd_wiki_lint` 的参数解析器只处理空格分隔格式：

```python
if args[i] == "--check" and i + 1 < len(args):
    check_mode = args[i + 1]
```

`"--check=stale"` 无法匹配 `args[i] == "--check"`，因此 `check_mode` 保持默认值 `"all"`，
导致即使指定 `check="stale"` 也会同时运行 orphan 检查。

## 修复方案

更新 `cmd_wiki_lint` 的参数解析器，同时支持两种格式：
- `--check stale`（空格分隔，两个 token）
- `--check=stale`（等号分隔，单个 token）

```python
if args[i].startswith("--check="):
    check_mode = args[i].split("=", 1)[1]
    i += 1
elif args[i] == "--check" and i + 1 < len(args):
    check_mode = args[i + 1]
    i += 2
```
