# Spec Kit Baseline

接受 Spec Kit 风格的 spec-first 开发约束。

## 规则

1. 复杂变更先写 spec / 目标 / 验收标准
2. 再拆 execution plan 和 task graph
3. 最后落代码、测试、文档

## 对 Agents-Memory 的直接要求

1. 新命令、新 profile、新 validation 层能力，优先先写设计文档
2. 验收标准必须可被测试或命令验证
